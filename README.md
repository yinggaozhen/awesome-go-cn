## 音频和音乐

*用于操作音频的库。 (翻译出错了? 试试 [英文版](README_EN.md#audio-and-music) 吧~)*

* [flac](https://github.com/mewkiz/flac)  原生 Go FLAC编码器/解码器，支持FLAC流。
* [gaad](https://github.com/Comcast/gaad)  原生 Go AAC位流解析器。
* [go-sox](https://github.com/krig/go-sox)  libsox 的 Go 语言实现接口。
* [GoAudio](https://github.com/DylanMeeus/GoAudio)  本地Go音频处理库。
* [gosamplerate](https://github.com/dh1tw/gosamplerate)  libsamplerate 的 Go 语言实现接口。
* [id3v2](https://github.com/bogem/id3v2)  ID3解码和编码库为Go。
* [malgo](https://github.com/gen2brain/malgo)  迷你音频库。
* [minimp3](https://github.com/tosone/minimp3)  轻量级MP3解码器库。
* [mix](https://github.com/go-mix/mix)  基于序列的 Go 原生音乐混音器。
* [mp3](https://github.com/tcolgate/mp3)  原生 Go MP3解码器。
* [music-theory](https://github.com/go-music-theory/music-theory)  基于 Go 的音乐理论模型。
* [Oto](https://github.com/hajimehoshi/oto)  多平台的 low-level 声音播放库。
* [PortAudio](https://github.com/gordonklaus/portaudio)  基于 Go 的PortAudio audio I/O库。
* [portmidi](https://github.com/rakyll/portmidi)  PortMidi的 Go 语言实现接口.
* [vorbis](https://github.com/mccoyst/vorbis)  “原生” Go Vorbis解码器(使用CGO，但没有依赖关系)。
* [waveform](https://github.com/mdlayher/waveform)  通过音频流生成波形图像的包。

## 身份验证和OAuth

*用于实现验证方案的库。 (翻译出错了? 试试 [英文版](README_EN.md#authentication-and-oauth) 吧~)*

* [authboss](https://github.com/volatiletech/authboss)  web模块化认证系统。它试图删除尽可能多的模板文件和硬编码，以便每次新建一个新的web项目时，您都可以插入、配置并开始构建您的应用程序，而不必每次都构建一个身份验证系统。
* [jwt-go](https://github.com/dgrijalva/jwt-go)  JSON Web令牌(JWT)。
* [sessionup](https://github.com/swithek/sessionup)  简单但有效的HTTP会话管理和标识包。
* [sessions](https://github.com/adam-hanna/sessions)  非常简单，高性能，可深度定制的会话服务，主要用于的 go http 服务器。
* [sessiongate-go](https://github.com/f0rmiga/sessiongate-go)  使用SessionGate Redis模块进行会话管理。
* [session](https://github.com/icza/session)  web服务器会话管理(包括支持谷歌应用程序引擎 - GAE)。
* [securecookie](https://github.com/chmike/securecookie)  高效安全的cookie编码/解码。
* [scs](https://github.com/alexedwards/scs)  HTTP服务器的会话管理器。
* [scope](https://github.com/SonicRoshan/scope)  在Go中轻松管理OAuth2范围。
* [rbac](https://github.com/zpatrick/rbac)  最小的RBAC包。
* [permissions2](https://github.com/xyproto/permissions2)  用于跟踪用户、登录状态和权限的库。依赖于cookie安全和bcrypt。
* [paseto](https://github.com/o1egl/paseto)  平台无关的安全令牌(PASETO)。
* [otpgo](https://github.com/jltorresm/otpgo)  基于时间的一次性密码(TOTP)和基于hmac的一次性密码(HOTP)库用于Go。
* [osin](https://github.com/openshift/osin)  OAuth2服务器库。
* [oauth2](https://github.com/golang/oauth2)  goauth2的继任者。通用OAuth 2.0包，附带JWT、谷歌api、计算引擎和应用程序引擎支持。
* [loginsrv](https://github.com/tarent/loginsrv)  JWT登录微服务带有可插拔的后端服务，如OAuth2 (Github)、htpasswd、osiam。
* [jwt-auth](https://github.com/adam-hanna/jwt-auth)  JWT中间件，可用于Golang http服务器，提供了许多配置选项。
* [branca](https://github.com/hako/branca)  基于 Go 实现Branca令牌。
* [jwt](https://github.com/cristalhq/jwt)  安全、简单、快速的JSON Web令牌。
* [jwt](https://github.com/pascaldekloe/jwt)  轻量级JSON Web令牌库。
* [jwt](https://github.com/robbert229/jwt)  简单易用的JSON Web令牌实现(JWT)。
* [jeff](https://github.com/abraithwaite/jeff)  简单、灵活、安全和惯用的web会话管理，具有可配置化的后端。
* [httpauth](https://github.com/goji/httpauth)  HTTP身份验证中间件。
* [goth](https://github.com/markbates/goth)  提供了 OAuth 和 OAuth2 的简单清晰易用的方法。可开箱即用处理多个提供程序。
* [gorbac](https://github.com/mikespook/gorbac)  轻量级的基于角色的访问控制(RBAC)实现。
* [gologin](https://github.com/dghubble/gologin)  用于使用OAuth1和OAuth2身份验证提供者登录的可链处理程序。
* [go-oauth2-server](https://github.com/RichardKnop/go-oauth2-server)  用 Golang 编写的独立且符合规范的OAuth2服务器。
* [go-jose](https://github.com/square/go-jose)  相当完整地实现了JOSE工作组的JSON Web令牌、JSON Web签名和JSON Web加密规范。
* [go-guardian](https://github.com/shaj13/go-guardian)  Go-Guardian是一个golang库，它提供了一种简单、干净和习惯的方式来创建强大的现代API和web身份验证，支持LDAP、Basic、承载令牌和基于证书的身份验证。
* [go-email-normalizer](https://github.com/dimuska139/go-email-normalizer)  Golang库，用于提供电子邮件地址的规范表示。
* [cookiestxt](https://github.com/mengzhuo/cookiestxt)  提供cookie .txt文件格式的解析器。
* [casbin](https://github.com/hsluoyz/casbin)  支持ACL、RBAC、ABAC等访问控制模型的授权库。
* [sjwt](https://github.com/brianvoe/sjwt)  简单的jwt生成器和解析器。
